Overlay Frame Generator GUIDELINES
Tool Introduction

Overlay Frame Generator

ライブ配信の画面構成において、ゲーム画面やカメラ映像を囲む「枠（フレーム）」や、お知らせを流す「テロップ」は、配信のクオリティを決定づける重要な視覚要素です。

しかし、従来の静的なPNG画像素材では、「配信解像度を変更した際に線がボケる」「告知内容やSNSアカウントを変更するたびに画像編集ソフトを立ち上げ直す必要がある」といった運用の煩わしさがありました。

「Overlay Frame Generator」は、これらの課題をコードの知識なしで解決できる配信特化型の動的HTMLオーバーレイ生成ツールです。SVGによる美しい可変フレームと、HTMLならではの柔軟なテキスト演出を融合し、OBSに読み込ませるだけの高機能な配信レイヤーを瞬時にエクスポートします。

スタンドアロン＆完全ローカル：
生成されたHTMLファイルは外部サーバーとの通信を行わず、ローカル環境（file:// プロトコル）で100%スタンドアロン動作します。配信中のネットワーク帯域を圧迫せず、セキュリティ面でも極めて安全な設計です。

01. 実用的な機能・自由なレイアウト思想

単なる「画面の飾り」に留まらない、配信者の実用性と自由度を最優先したアドオン設計が施されています。

枠なし・パーツ単体運用が可能

各要素は完全に独立してオン/オフを切り替え可能。メインフレームすらオフにすることで、「流れるテロップだけ」「自動巡回するQRコードだけ」といった単体パーツとしてHTMLを出力できます。

自作の透過PNG枠との連携

すでにデザインされたお気に入りの透過画像（PNG等）をお持ちの場合、その上から本ツールで出力したHTMLを重ねるだけで、既存のデザインを活かしたまま動的な時計やテロップの機能を追加できます。

劣化のないSVGベクターフレーム

メイン枠は解像度に依存しないSVGコードで出力。FHD、縦型ショート動画、4Kなど、どのキャンバスサイズに変更しても線幅やネオングロウが一切劣化せず美しさを保ちます。

複数お知らせ対応ティッカー

最大3つまで登録できる告知テロップは、1周スクロールするごとに自動で次のテキストへ滑らかに切り替わるループアニメーション機構を内蔵しています。

ロゴ・グラデーション対応QRローテーター

複数の告知URLを、指定秒数ごとにフェードやスライドで巡回表示。各ブランドロゴの埋め込みや、美しいグラデーションカラーの背景マスキングもCSSブレンドモードで高度に処理されています。

02. 開発環境・技術仕様

出力されるHTMLファイルは一切の依存関係（外部JSライブラリ等）を持たず、純粋なネイティブJavaScript、CSSアニメーション、およびインラインSVGのみで構築されます。

UI Architecture

Tailwind CSS ＆ Vanilla Data State

ジェネレーター自体はフロントエンドをTailwind CSS、ロジックを直感的なVanilla JSのステートオブジェクト（`state`）で駆動させ、リアルタイムなDOM更新とプレビュー倍率スケーリングを軽量に実現しています。

Output Engine

Blob URL ＆ iframe srcdoc

プレビュー内のカスタム時計HTML表示には `Blob URL` 構造を採用し、エクスポート時には自己完結型の `srcdoc` 属性へとシリアライズ。OBS上で安全かつ正確なクロスドメイン隔離を行います。

03. 出力例と異なるレイアウトデモ

本ツールによって出力されるHTMLレイヤーの実際の挙動を示す、2つの異なるコンセプトのデモサンプルです。

DEMO 01: フルコンプリート・セットアップ 16:9 Canvas

等倍フルレイヤー例： 外枠フレーム、ヘッダータイトル、SNS、テロップ、ワイプ、および左下のQRコードがすべての調和を保って連動している状態です。

DEMO 02: 独立パーツ化（単体ウィジェット・変則自由配置例） Component Widgets

単体ウィジェット・変則配置例： メインの「外枠フレーム」を非表示に設定し、各アドオンを完全に独立した「単体ウィジェット（独立装置）」として切り出した状態です。画面中央付近にQRコードをぽつんと浮かせるなど、ジェネレーターの定型枠に縛られない自由なレイアウト運用が可能です。自作のPNG透過枠と組み合わせる際も、この方法で必要な機能パーツだけを画面のベストポジションへ浮遊させることができます。

04. OBS Studioへの導入と配置テクニック

ジェネレーターから出力されたHTMLファイルをOBSに組み込む手順と、設定作業を高速化するTipsです。

方法 1 ソース追加による確実なピクセル配置（推奨）

一番確実かつ正確に位置決めを行う王道の追加方法です。

  1. OBSの「ソース」エリアの「＋」をクリックし、「ブラウザ」を選択。
  2. 「ローカルファイル」にチェックを入れ、「参照」からエクスポートした obs_overlay.html を選択。
  3. ジェネレーターで指定した解像度と同じ値（例：幅 1920 、高さ 1080）をプロパティに入力。
  4. 「カスタムCSS」欄の記述をすべて消去し、完全に空にする（背景透過に必須）。

裏技 プレビュー画面へのダイレクト・ドラッグ＆ドロップ

手早くレイアウトを確認したい時や、厳密な配置を求めないパーツ単体運用の時に有効なショートカットトリックです。

PCに保存した obs_overlay.html ファイルを、OBSのプレビュー画面（配信画面が映っているエリア）へ直接マウスでドラッグ＆ドロップしてください。

これだけで、OBSが自動的にブラウザソースを作成し、キャンバスサイズにフィット（オートフィット）させた状態で一発配置してくれます。追加した後は自動で背景も透過されます。

※ただし、キャンバス比率や解像度がプロパティ側で自動追従しないケースがあるため、ミリ単位での厳密な位置合わせが必要な場合は、通常の「方法1（ソース追加）」から幅・高さを数値指定して配置を行ってください。

05. まとめ

「Overlay Frame Generator」は、従来の画像ベースの枠素材が持っていた「固定化」「編集の難しさ」という制約を取り払い、Web標準技術を用いて配信画面をコードレスに動的化するユーティリティです。

フレームとしての美しさを追求するフルセット運用から、自作PNG枠と重ね合わせるアドオン運用、さらにはパーツ単体をドラッグ＆ドロップで配置するミニマル運用まで、配信スタイルに合わせた極めて自由な画面設計をアシストします。

本ツールによって、あなたの配信環境がよりスマートでメンテナンス性の高いものへと進化することを願っています。
